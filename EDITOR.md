# 课文编辑器（临时工具）

一个脱离主站点、专门用来批量修改课文内容的本地编辑页面。保存时直接改写 `src/data/books.ts`，Next.js 热重载会让主站立即生效。

## 启动

```bash
npm install        # 首次或换机器后
npm run dev
```

打开 **http://localhost:3000/editor**（若 3000 被占用会自动用 3001，终端里 `Local:` 会提示实际端口）。

## 页面结构

- **左侧**：3 本书折叠列表，点书名展开课文
- **每一课**：显示标题、卡片数量、「编辑」按钮、「删除」按钮
- **右侧**：编辑面板 — 标题输入框 + 两个大文本框（左法文、右中文）

## 编辑规则

- **一行一张卡片**。法文文本框第 N 行 ↔ 中文文本框第 N 行
- 空行会被忽略
- 两侧行数不一致时，顶部会显示红色警告。保存仍然可行，短的一侧会补空字符串
- 保存后卡片 id 会重新生成（`c1`, `c2`, ...），localStorage 里已掌握的进度会失效 — 这是**有意为之**，因为编辑内容后旧的 id 也对应不上新卡片了

## 文件写入行为

- 保存只改写 `src/data/books.ts`
- 格式：双引号 JSON 字符串（原本是模板字符串反引号）— 功能等价，TypeScript 两种都认
- 每次保存都会重新格式化整个文件。手工编辑过的风格会丢失
- **先用 git 提交当前状态，再在编辑器里改** — 万一序列化出问题可以随时 `git checkout src/data/books.ts` 回滚

## 架构

```
src/app/editor/page.tsx          客户端 UI（树 + 编辑面板）
src/app/api/editor/route.ts      GET / PUT / DELETE，Node runtime，直接 fs.writeFile
src/data/books.ts                数据源；API 读它、写它
```

- API 用进程内 `state` 变量缓存数据，避免多次请求间因 HMR 未就绪而读到旧值
- `force-dynamic` 确保 GET 不被缓存
- `runtime = "nodejs"` 因为要用 `fs`

## 还没做的事

- 新增课 / 新增书：目前只能编辑和删除已有的
- 重命名书、换书的图标、调整课的顺序：都没有 UI，需要直接改 `books.ts`
- 生产模式（`npm run build && npm start`）下 API 能跑，但没做鉴权 — **仅限本地使用**

## 工作流建议

1. 进编辑器，找到要改的课，点「编辑」
2. 把两个文本框整体替换成新的法文 / 中文
3. 检查两侧行数一致
4. 点保存，等终端显示热重载完毕
5. 去主站（`/`）翻到那一课验证一下
6. 一批改完后，`git add -A && git commit`

## 已知差异

初次提交后，`books.ts` 会被编辑器改写成 JSON 双引号格式（原始提交是反引号模板字符串）。第一次保存时的 diff 会非常大但是语义上零改动，之后的 diff 就只反映真实内容变化了。
